# 当前进度

- 当前正在做：继续推进 phase2，把工作台从“能配一部分”推进到“更完整可配置”，同时继续收紧长期运行、告警和人工接管。这轮先把浏览器级前端验证链恢复，再继续往更完整配置能力推进。
- 上次停留位置：评估页已经能看到最近两轮变化和研究/执行差异，任务页也已补上同步失败细节和人工接管时间线，但浏览器自动化测试还被 `snap chromium` 环境卡住。
- 近期关键决定和原因：
  - 工作台配置继续只走统一配置中心，不再给数据、特征、研究、回测、自动化各开一套保存逻辑，避免配置漂移。
  - 研究结果和当前配置是否一致，统一通过 `config_alignment` 和“最近两轮对比”显性化，避免只看到结论、看不到上下文。
  - 研究训练、推理、流水线统一改成后台运行，并显式给出阶段、进度、预计时长和结果去向，避免“点了按钮不知道系统在干什么”。
  - 如果实验账本缺少配置快照，页面必须明确显示“暂时无法比较”，不能再误写成“没有变化”。
  - 自动化健康摘要现在必须把同步失败和人工接管分开讲清楚，所以后端统一补了 `last_failure_at / last_sync_failure_at / paused_since / takeover_since / sync_failure_count`，前端直接按这套字段渲染。
  - 长期运行能力这轮继续保持“先把真实状态讲清楚，再放更多自动化动作”，所以任务页优先展示同步失败细节、人工接管时间线、长期运行参数和执行安全门，而不是继续堆按钮。
  - Playwright 浏览器测试现在不再硬编码 `/snap/bin/chromium`，统一改成优先使用 Playwright 自带浏览器；如果以后要强制指定浏览器，走 `PLAYWRIGHT_EXECUTABLE_PATH` 环境变量。
  - 这台机器缺少 Playwright Chromium 运行库时，直接用 `pnpm exec playwright install-deps chromium` 补齐系统依赖；浏览器级前端测试现在已经能真实跑通。
  - 浏览器测试登录这轮改成走真实登录页，不再依赖不存在的旧登录 API；后续前端关键交互验证统一优先走真实页面链路。
