# 当前进度

- 当前正在做：继续推进 phase2，把工作台从“能配一部分”推进到“更完整可配置”，同时继续收紧长期运行、告警和人工接管；这轮先收掉了市场页卡住、信号页动作校验不稳、前端样式失效这几类稳定性问题。
- 上次停留位置：数据、特征、研究、回测、评估、执行与自动化这 6 个工作台已经显性化，并且第一批配置项已经能保存和生效；最近两轮实验对比也已经能在评估页查看。
- 近期关键决定和原因：
  - 研究页现在显式开放持有窗口选择；回测页显式开放成本模型选择；评估页显式开放规则门、验证门、回测门、一致性门和 live 门开关，避免后端已经支持、前端仍然看不见。
  - 前端 fallback 数据必须跟真实类型同步补齐；否则生产构建会在类型检查阶段失败，看起来像页面问题，实际是兜底模型缺字段。
  - 浏览器测试默认端口这轮统一切到 `9021/9022`，这样本地调试和真实页面验证不再每次都手动带环境变量。
  - 工作台配置继续只走统一配置中心，不再给数据、特征、研究、回测、自动化各开一套保存逻辑，避免配置漂移。
  - 研究结果和当前配置是否一致，统一通过 `config_alignment` 和“最近两轮对比”显性化，避免只看到结论、看不到上下文。
  - 研究训练、推理、流水线统一改成后台运行，并显式给出阶段、进度、预计时长和结果去向，避免“点了按钮不知道系统在干什么”。
  - 如果实验账本缺少配置快照，页面必须明确显示“暂时无法比较”，不能再误写成“没有变化”。
  - 自动化健康摘要现在必须把同步失败和人工接管分开讲清楚，所以后端统一补了 `last_failure_at / last_sync_failure_at / paused_since / takeover_since / sync_failure_count`，前端直接按这套字段渲染。
  - 长期运行能力这轮继续保持“先把真实状态讲清楚，再放更多自动化动作”，所以任务页优先展示同步失败细节、人工接管时间线、长期运行参数和执行安全门，而不是继续堆按钮。
  - Playwright 浏览器测试现在优先使用 Playwright 自带浏览器，并通过 `QUANT_WEB_BASE_URL / QUANT_API_BASE_URL` 适配调试端口；前端关键交互验证恢复可用。
  - 单币页图表数据如果基础 K 线为空，现在直接按空态返回，不再继续串行拉多周期摘要；这样 `/market/[symbol]` 不会因为上游无数据而整页卡住。
  - 信号页和登录页的浏览器测试现在拆成“两层”：一层只测运行中反馈，直接阻止表单离开当前页；另一层再测最终跳转结果，避免导航时序把真实交互误报成失败。
  - 表单按钮的运行中反馈现在同时覆盖点击提交和 Enter 提交；表单开始提交时就会进入运行中，不再只靠按钮点击分支。
  - 表单按钮的提交改成先同步渲染运行中，再延迟触发表单提交，避免按钮节点切换时触发浏览器异常。
  - 前端重新构建后必须重启 `next start`，否则旧进程会继续引用失效 chunk，页面会退化成纯文字；这轮已经按这个规则重启了 `9022`。
