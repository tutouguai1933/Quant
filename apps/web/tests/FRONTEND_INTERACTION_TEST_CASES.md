# 前端界面交互测试用例

## 页面功能职责总览

| 分类 | 页面 | 路径 | 功能职责 |
|------|------|------|----------|
| 决策驾驶舱 | 首页 | `/` | 快速决策入口，显示当前状态和下一步动作 |
| 决策驾驶舱 | 信号页 | `/signals` | 候选判断 + 研究报告，运行研究流水线 |
| 决策驾驶舱 | 评估页 | `/evaluation` | 哪些候选值得推进、为什么推荐/淘汰 |
| 决策驾驶舱 | 策略页 | `/strategies` | 策略状态、执行判断、派发动作 |
| 决策驾驶舱 | 任务页 | `/tasks` | 自动化监控与恢复操作 |
| 研究配置 | 研究页 | `/research` | 研究配置和产物查看 |
| 研究配置 | 因子页 | `/features` | 因子体系、权重配置 |
| 研究配置 | 数据页 | `/data` | 数据来源、快照状态 |
| 研究配置 | 回测页 | `/backtest` | 回测结果、成本模型 |
| 市场交易 | 市场页 | `/market` | 市场行情筛选 |
| 市场交易 | 单币页 | `/market/[symbol]` | 图表 + 研究判断 |
| 账户工具 | 余额页 | `/balances` | 账户资产核对（不承担判断） |
| 账户工具 | 订单页 | `/orders` | 执行回报核对（不承担判断） |

| 账户工具 | 风险页 | `/risk` | 告警规则核对（不承担判断） |

## 1. 首页 `/`

### 功能职责
- 快速决策入口，显示当前状态和下一步动作
- 集成研究状态、执行状态、风险告警三块摘要

### 测试用例

#### TC-HOME-001: 状态栏显示
- **预期**: 状态栏显示 4 个状态项（信号总数、研究状态、自动化、最新状态）
- **验证**: 每个状态项有 label、value、status、detail

#### TC-HOME-002: 研究状态卡片
- **预期**: 显示研究运行状态面板
- **交互**: 
  - 点击"运行 Qlib 信号流水线" → 触发研究动作 → 显示进度条
  - 点击"查看研究报告" → 跳转到 `/signals`

#### TC-HOME-003: 执行状态卡片
- **预期**: 显示策略执行摘要
- **交互**: 
  - 点击"启动策略" → 触发策略启动
  - 点击"派发最新信号" → 触发信号派发
  - 点击"去策略页确认执行" → 跳转到 `/strategies`

#### TC-HOME-004: 风险与告警卡片
- **预期**: 显示自动化状态摘要
- **交互**: 
  - 点击"去任务页处理自动化" → 跳转到 `/tasks`

#### TC-HOME-005: 未登录状态
- **预期**: 显示登录提示，跳转按钮带登录重定向
- **验证**: 点击执行入口 → 跳转到 `/login?next=/strategies`

## 2. 信号页 `/signals`

### 功能职责
- 左侧：候选排行榜 + 研究动作
- 右侧：统一研究报告 + 最近实验摘要

### 测试用例

#### TC-SIGNALS-001: 状态栏显示
- **预期**: 状态栏显示 4 个状态项（信号总数、研究状态、自动化、最新状态）
- **验证**: 研究状态显示候选数和可进入 dry-run 数

#### TC-SIGNALS-002: 运行研究流水线
- **交互**: 
  - 点击"运行 Qlib 信号流水线" → 触发后台任务 → 进度条显示
  - WebSocket 实时推送进度更新
- **验证**: 研究运行状态面板显示当前阶段、进度百分比

#### TC-SIGNALS-003: 候选排行榜
- **预期**: 显示候选总数、可进入 dry-run 数、筛选通过率
- **验证**: 每个候选显示：排名、币种、分数、策略模板、门控状态、失败原因

#### TC-SIGNALS-004: 候选筛选切换
- **交互**: 
  - 点击"全部" → 显示所有候选
  - 点击"可进 dry-run" → 只显示 allowed_to_dry_run 的候选
  - 点击"继续研究" → 只显示 next_action=continue_research 的候选
  - 点击"强制验证" → 只显示 forced_for_validation 的候选

#### TC-SIGNALS-005: 门控视图切换
- **交互**: 
  - 点击"全部门" → 显示所有候选
  - 点击"规则门" → 只显示规则门失败的候选
  - 点击"回测门" → 只显示回测门失败的候选
  - 点击"验证门" → 只显示验证门失败的候选

#### TC-SIGNALS-006: 研究运行历史
- **预期**: 显示 training/inference/pipeline 的最近运行次数和平均耗时
- **验证**: 有历史记录时显示"最近 N 次，平均 X 秒"

#### TC-SIGNALS-007: 自动化入口
- **预期**: 显示自动化摘要和下一步动作
- **交互**: 点击"去任务页看自动化" → 跳转到 `/tasks`

#### TC-SIGNALS-008: 研究动作按钮
- **交互**: 
  - 点击"研究训练" → 触发训练任务
  - 点击"研究推理" → 触发推理任务

## 跨页面流程测试用例

### TC-FLOW-001: 研究到执行完整流程
1. 首页点击"运行 Qlib 信号流水线"
2. 等待训练完成（进度条 100%）
3. 信号页查看候选排行榜
4. 评估页查看推荐标的
5. 策略页确认执行
6. 任务页查看自动化状态

### TC-FLOW-002: 未登录到登录流程
1. 首页点击策略入口
2. 跳转到登录页
3. 输入账号密码登录
4. 跳转到策略页

### TC-FLOW-003: 工具页返回流程
1. 从策略页点击"余额"
2. 余额页显示数据
3. 点击"回到策略页"返回

## 状态一致性测试用例

### TC-CONSISTENT-001: 仲裁结论一致性
- **验证**: 首页、评估页、策略页、任务页的仲裁结论保持一致
- **检查**: 当前状态、下一步动作字段相同

### TC-CONSISTENT-002: 候选数据一致性
- **验证**: 信号页候选数 = 评估页候选总数
- **检查**: candidate_count 字段相同

### TC-CONSISTENT-003: 执行器状态一致性
- **验证**: 策略页执行器状态 = 订单页/持仓页同步来源
- **检查**: 执行器连接状态一致
