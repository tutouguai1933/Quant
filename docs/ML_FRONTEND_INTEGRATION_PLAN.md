# 前端页面与 ML 系统联动优化方案

> 版本：v1.0  
> 日期：2026-05-13  
> 状态：规划中

---

## 一、背景与目标

### 1.1 当前问题

系统已完成 ML 基础设施建设（模型注册表、超参数优化、自动重训练），但前端页面与自动化周期 ML 策略之间存在断层：

| 问题类型 | 具体表现 |
|---------|---------|
| **训练流程断裂** | `/training` 页面显示的是独立训练结果，自动化周期的训练任务结果不可见 |
| **推理流程不透明** | `/signals` 页面不显示 ML 模型预测分数和特征贡献 |
| **模型管理缺失** | 没有统一的模型版本管理界面 |
| **回测与ML脱节** | `/backtest`、`/evaluation` 不使用 ML 模型 |
| **因子分析独立** | `/features` 页面与 ML 特征工程无关联 |

### 1.2 优化目标

1. **可视化闭环**：用户可在前端看到完整的 ML 训练→推理→评估闭环
2. **数据一致性**：前端展示的数据与自动化周期实际使用的数据一致
3. **可追溯性**：每个候选币种的评分来源可追溯到具体模型版本
4. **易用性**：降低理解成本，用户无需理解后端架构即可使用

### 1.3 涉及页面

| 页面 | 路径 | 优先级 |
|------|------|--------|
| 模型训练 | `/training` | P0 |
| 模型管理 | `/models`（新增） | P0 |
| 信号 | `/signals` | P1 |
| 选币回测 | `/evaluation` | P2 |
| 因子研究 | `/features` | P2 |
| 参数优化 | `/hyperopt` | P2 |
| 回测训练 | `/backtest` | P3 |

### 2.2 核心数据模型

#### 模型注册记录 (MLModelRecord)

```typescript
interface MLModelRecord {
  version_id: string;           // 模型版本ID
  model_type: string;           // lightgbm / xgboost
  created_at: string;           // 创建时间
  training_duration_seconds: number;
  metrics: {
    train_auc: number;
    validation_auc: number;
    train_f1?: number;
    validation_f1?: number;
  };
  feature_importance: Array<{
    feature: string;
    importance: number;
  }>;
  training_curve: Array<{
    step: number;
    train_score: number;
    validation_score: number;
  }>;
  stage: "development" | "staging" | "production";
  tags: string[];
  source: "manual" | "automation_cycle" | "hyperopt";
  parent_version_id?: string;
}
```

#### 信号记录增强 (Signal)

```typescript
interface Signal {
  // 现有字段...
  symbol: string;
  score: number;
  
  // 新增 ML 相关字段
  ml_prediction?: {
    probability: number;        // ML 模型预测的正类概率
    model_version: string;      // 使用的模型版本
    feature_contributions: Array<{
      feature: string;
      value: number;
      contribution: number;     // SHAP 值或类似贡献度
    }>;
  };
}
```

### 3.4 `/evaluation` 页面优化

#### 当前问题
- 候选评估不使用 ML 模型
- 回测结果与 ML 预测无关联

#### 优化方案

**新增区块**：
1. **ML 预测概览**：显示每个候选的 ML 预测分数
2. **特征分布**：候选币种的特征值分布图
3. **预测置信度**：ML 模型对每个候选的置信度分析

**后端API调整**：
- 修改 `GET /evaluation/workspace` - 增加 ML 相关字段
- 修改 `POST /evaluation/run` - 使用 ML 模型评分

### 3.5 `/features` 页面优化

#### 当前问题
- 因子分析与 ML 特征工程独立
- 不显示 ML 模型实际使用的特征重要性

#### 优化方案

**新增区块**：
1. **生产模型特征重要性**：从当前生产模型读取
2. **特征-预测关联**：分析特征值与预测分数的相关性
3. **特征漂移检测**：监控特征分布变化

**后端API调整**：
- 新增 `GET /ml/features/importance` - 获取生产模型特征重要性
- 新增 `GET /ml/features/drift` - 特征漂移分析

### 3.6 `/hyperopt` 页面优化

#### 当前问题
- 使用独立的 hyperopt 路由
- 自动化周期的 ML hyperopt 结果不可见

#### 优化方案

**统一到 ML hyperopt**：
- 将旧的 `/hyperopt` 路由标记为 deprecated
- 页面调用 `/ml/hyperopt/*` 系列接口
- 显示自动化周期触发的优化任务

## 六、易用性设计

### 6.1 信息层次

| 层次 | 内容 | 位置 |
|------|------|------|
| L1 概览 | 模型总数、生产模型版本、最近训练状态 | 页面顶部卡片 |
| L2 详情 | 训练曲线、特征重要性、模型列表 | 页面中部 |
| L3 深入 | 特征贡献、训练日志、版本对比 | 折叠/弹窗 |

### 6.2 状态指示

| 状态 | 颜色 | 说明 |
|------|------|------|
| 生产中 | 绿色 ● | 当前正在使用的模型 |
| 备用 | 蓝色 ○ | 可切换的模型 |
| 开发中 | 灰色 ◌ | 测试阶段的模型 |
| 训练中 | 黄色 ◐ | 正在训练 |

### 6.3 操作引导

- **首次访问**：显示引导提示，解释页面功能
- **空状态**：显示"暂无数据"和"如何触发训练"说明
- **错误状态**：显示具体错误信息和解决建议
- **加载状态**：骨架屏 + 加载进度

### 6.4 响应式设计

| 屏幕尺寸 | 布局调整 |
|---------|---------|
| 桌面 (>1200px) | 三栏卡片 + 图表并排 |
| 平板 (768-1200px) | 两栏卡片 + 图表堆叠 |
| 手机 (<768px) | 单栏 + 折叠图表 |

## 七、性能优化设计

### 7.1 前端优化

| 优化项 | 方案 |
|--------|------|
| 数据缓存 | React Query 缓存模型列表，5 分钟刷新 |
| 图表懒加载 | 使用 Intersection Observer 延迟加载图表 |
| 分页加载 | 模型历史列表支持分页，默认 20 条/页 |
| 防抖节流 | 搜索、筛选操作防抖 300ms |

### 7.2 后端优化

| 优化项 | 方案 |
|--------|------|
| API 缓存 | `/ml/models` 列表缓存 60s |
| 数据预聚合 | 特征重要性预计算，存入模型元数据 |
| 批量查询 | 训练历史支持批量获取多个版本 |
| 异步加载 | 大型图表数据异步加载 |

### 7.3 推理优化

| 优化项 | 方案 |
|--------|------|
| 模型预加载 | 生产模型启动时加载到内存 |
| 批量推理 | 多个候选币种批量预测 |
| 结果缓存 | 推理结果缓存到信号快照 |
