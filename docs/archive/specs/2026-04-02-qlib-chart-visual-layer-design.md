# Qlib 图表图层与研究文案统一设计

## 背景判断

统一研究驾驶舱已经把市场页、单币页和策略页接到同一组研究字段上，但还有两个明显缺口：

- 单币页现在主要还是文字说明，信号点、入场位、止损位没有形成直观看图层
- 三个页面虽然用了同一批字段，但文案顺序和强调重点还不一致

这会让用户知道“系统怎么看”，但还不容易一眼看出“图上关键位置在哪”。所以这一步继续沿用现有页面，不新开页面，只把图表可读性和页面口径收紧。

## 总体结论

推荐方案固定为：

- **保留现有最小图表组件，补轻量图表图层和统一研究摘要区**

也就是：

- 不引入新的图表依赖
- 继续使用当前 `CandleChart`
- 把最近信号、入场位、止损位做成图表旁的直观图层摘要
- 同时把市场页、单币页、策略页统一成同一套研究说明语言

## 设计目标

这一步只回答 4 个问题：

- 图上现在最重要的点位在哪里
- 研究层当前是在支持、压低还是观望
- 当前更适合哪套策略
- 三个页面是不是都在用同一套说法

## 推荐方案与备选方案

### 方案 1：轻量图表图层 + 文案统一

这是推荐方案。

做法：

- `CandleChart` 增加轻量图层摘要区
- 单币页把研究结论、主原因、信号点、入场位、止损位按统一顺序展示
- 市场页和策略页改成同一组标签

优点：

- 改动小
- 不引入新依赖
- 适合当前多 session 并行
- 能满足“先看图上关键位置”的目标

缺点：

- 视觉表达还不是完整交互式图表

### 方案 2：接入复杂图表库

当前阶段不推荐。

问题：

- 范围明显变大
- 会碰到更多前端和依赖变化
- 不适合现在的收口节奏

### 方案 3：只统一文案

当前阶段不推荐。

问题：

- 页面说法会更整齐
- 但图上关键位置还是不够直观
- 不能完成这一轮核心目标

## 页面设计

### 1. 市场页

市场页继续做“先筛选再点进去”的入口，但文案统一成同一组概念：

- 研究倾向
- 推荐策略
- 判断信心
- 主判断

这里不展开完整解释，只保留快速筛选所需的信息。

### 2. 单币页

单币页是这一步的主页面。

页面结构调整为：

- 图表摘要
- 图表图层摘要
- 当前判断
- 研究解释
- Freqtrade 准备情况

其中“图表图层摘要”负责把以下信息放在一起：

- 最近信号点数量
- 最近入场位
- 最近止损位
- 研究倾向
- 研究门控
- 判断信心

不直接在图上画复杂交互层，而是用和图表相邻的结构化信息，把“图上的关键位置”读出来。

### 3. 策略页

策略页继续保留策略卡片，但每张卡片的信息顺序统一为：

- 运行状态
- 当前判断
- 推荐策略
- 研究倾向
- 判断信心
- 研究门控
- 主判断
- 研究解释

这样用户从市场页到单币页再到策略页，不会反复看到不同词。

## 数据与契约

这一步尽量复用现有契约：

- 市场页继续使用 `research_brief`
- 单币页继续使用 `research_cockpit` 和 `markers`
- 策略页继续使用 `research_cockpit`

只有在前端渲染需要时，才补少量辅助字段；默认不扩大后端接口范围。

## 降级规则

这一步继续沿用前一轮统一降级规则：

- 没有研究结果：明确显示“暂无研究结论”
- 研究分数异常：按不可用处理
- 没有入场位或止损位：显示 `n/a`
- 没有信号点：显示 `0`
- 没有图表数据：继续显示现有空状态

## 实施边界

这次要做：

- 单币页补轻量图表图层摘要
- 市场页、单币页、策略页统一研究文案顺序和标签
- 补测试、review、真实页面验证、文档和提交

这次不做：

- 新增独立驾驶舱页面
- 引入新图表库
- 改软门控规则
- 新增执行链路

## 验证原则

至少覆盖：

- 前端测试能证明三页使用统一标签
- 单币页存在图表图层摘要和关键字段
- 类型检查和构建通过
- 真实页面能看到新增节点和统一文案
- 关键接口仍返回正确数据

## 对多 session 协同的影响

这一步仍然只收口 `Qlib` 研究展示层，不改变 `Freqtrade` 执行边界。

所以：

- 我这边继续负责研究图层和展示口径
- 另一个 session 继续负责执行链路
- 双方仍通过已稳定的统一研究字段衔接
