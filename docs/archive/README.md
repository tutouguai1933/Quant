# 文档归档说明

这里存放的是早期阶段性设计和实施计划。

这些文档的价值主要是：

- 追溯为什么当时这么做
- 回看阶段性目标和当时的边界
- 查某个旧功能是在哪一轮引入的

这些文档**不再作为当前开发入口**。  
新的会话不要先从这里开始，而是先读：

1. `CONTEXT.md`
2. `README.md`
3. `docs/roadmap.md`
4. `docs/architecture.md`
5. `docs/developer-handbook.md`

## 当前归档结构

- `docs/archive/specs/`
  - 历史设计文档
- `docs/archive/plans/`
  - 历史实施计划
- `docs/archive/notes/`
  - 已过期但仍保留参考价值的旧说明

## 为什么要归档

当前项目已经经历了多轮阶段性开发。  
如果把所有旧 spec / plan 都继续留在主路径里，新的会话会很容易被这些旧阶段目标干扰。

归档的目的不是删历史，而是：

- 保留追溯能力
- 收紧当前主入口
- 降低新 AI 接手时的信息噪音
