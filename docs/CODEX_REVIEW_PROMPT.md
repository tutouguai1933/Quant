# Quant 项目全面审阅提示词

## 审阅范围

对 Quant 项目进行全面审阅，重点关注以下方面：

### 1. 前端页面质量

**性能**
- 页面加载速度、首屏渲染时间
- 不必要的重复请求、数据缓存策略
- 组件懒加载、代码分割是否合理

**逻辑正确性**
- 页面跳转、导航链接是否指向正确目标
- 按钮功能与标签是否一致
- 状态显示是否准确（如连接状态、数据状态）
- 表单提交、数据刷新逻辑是否正常

**内容与可读性**
- 文案是否清晰易懂，术语是否恰当
- 空状态、错误状态是否有明确提示和引导
- 数据展示是否完整，字段命名是否一致

**界面设计**
- 布局是否合理，信息层级是否清晰
- 交互反馈是否及时（loading、success、error）
- 响应式适配是否正常
- 是否存在可优化的空间

### 2. 功能完整性

**核心功能验证**
- 检查各页面功能是否可正常使用
- API 调用是否正常返回数据
- 需要认证的页面是否有正确的权限控制

**页面与标题匹配**
- 面包屑、页面标题、侧边栏导航是否一致
- 页面内容是否符合页面名称和定位

### 3. 自动化运维能力

- 系统健康监控是否完善
- 告警推送是否正常工作
- 自动化巡检、恢复机制是否可用
- 运维操作入口是否便捷

### 4. 项目目标符合度

参考 `CONTEXT.md` 和 `docs/PROJECT_OVERVIEW.md` 了解项目定位，评估：
- 当前实现是否符合项目目标
- 功能设计是否满足量化交易控制平面需求
- 是否有明显的功能缺失或冗余

---

## 审阅方法

1. **访问线上环境**: http://39.106.11.65:9012
2. **逐页检查**: 从首页开始，逐一访问所有可访问页面
3. **功能测试**: 对关键功能进行交互测试
4. **使用 Playwright**: 对关键页面进行自动化测试，验证功能正确性
5. **记录问题**: 按优先级（高/中/低）分类记录发现的问题

## 注意事项

- 审阅应基于实际线上运行状态，而非代码静态分析
- 关注用户体验，从实际使用者角度评估
- 问题应具体可操作，避免模糊描述
- 优化建议应考虑实现成本和收益
