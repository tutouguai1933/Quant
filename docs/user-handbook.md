# Quant 使用手册

> 最后更新：2026-05-11

这份文档面向”怎么用系统”的问题。
如果你只想知道页面应该怎么点、结果应该去哪看，看这份就够了。

## 1. 先理解这套系统

这套系统不是一个”一键赚钱”的黑盒。
它现在更像一套个人量化工作台：

1. 先做研究
2. 再做评估
3. 再决定是不是进 `dry-run`
4. 最后才考虑小额 `live`
5. 每一轮都要复盘

## 3. 最推荐的使用动线

### 研究到执行

1. 进入 `/research`
2. 点 `研究训练`
3. 点 `研究推理`
4. 去 `/evaluation` 看推荐和淘汰原因
5. 去 `/strategies` 看当前推荐候选
6. 去 `/tasks` 看自动化和接管状态
7. 去 `/balances`、`/orders`、`/positions` 看执行结果

### 一键流水线

如果你想直接跑一整轮：

1. 去 `/signals`
2. 点 `运行 Qlib 信号流水线`
3. 看页面里的“研究运行状态”
4. 跑完后去 `/evaluation`
5. 再去 `/strategies` 和 `/tasks`

## 4. 各工作台怎么用

### 数据工作台 `/data`

这里主要看：

- 数据来源
- 研究标的
- 时间周期
- 时间范围
- 样本数量
- 数据快照

这里主要改：

- 研究池
- 主标的
- 固定日期 / 滚动窗口
- 样本长度

### 特征工作台 `/features`

这里主要看：

- 因子分类
- 主判断因子
- 辅助因子
- 类别权重

这里主要改：

- 趋势 / 动量 / 震荡 / 成交量 / 波动率权重
- 主判断因子
- 辅助因子
- 预处理方式

### 研究工作台 `/research`

这里主要看：

- 研究模板
- 模型
- 标签方式
- 标签预设
- 持有窗口
- 训练 / 验证 / 测试切分

这里主要改：

- 研究模板
- 模型
- 标签方式
- 触发口径
- 持有窗口
- 标签目标和止损

### 回测工作台 `/backtest`

这里主要看：

- 成本模型
- 净收益
- 回撤
- Sharpe
- 动作段

这里主要改：

- 手续费
- 滑点
- 成本模型
- `dry-run / live` 门槛
- 门控开关

### 评估与实验中心 `/evaluation`

这里主要看：

- 推荐原因
- 淘汰原因
- 最近两轮对比
- 研究 / 回测 / 执行差异

这里主要改：

- 准入预设
- 比较窗口
- 门控开关

### 执行和自动化 `/strategies`、`/tasks`

这里主要看：

- 当前推荐候选
- 当前执行模式
- 当前告警

- `Kill Switch`

## 6. 什么情况下应该停

这些情况先不要继续推进：

- 研究页显示当前结果和当前配置不一致
- 评估页显示主要阻塞还没解除
- 任务页有错误告警
- 任务页显示正在人工接管
- 当前候选只能进 `dry-run`，还不满足 `live`

## 7. 什么情况下可以继续

一般要同时满足：

- 研究结果已经跑完
- 评估页给出明确推荐
- 当前候选没有主要阻塞
- 任务页没有错误级告警
- 任务页没有处在人工接管中

## 8. 最重要的一句

当前系统的正确用法不是“先下单，再看发生了什么”，  
而是“先研究、先评估、先看任务页，再决定是不是继续执行”。
